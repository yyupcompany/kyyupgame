# 教学中心开发文档

## 1. 功能概述

教学中心是基于脑科学课程的教学管理平台，采用活动中心的UI样式，通过4个标签页管理不同的教学内容和进度跟踪。**新增班级图片视频支持和学员学习照片/视频记录功能，方便园长监督教学质量和学员完成情况。**

## 2. 数据结构分析

### 2.1 现有数据基础
- **班级数据**：`classes` 表包含班级基本信息
- **学生数据**：`students` 表包含学生基本信息
- **文件存储**：`file_storages` 表支持图片、视频等媒体文件存储
- **文件上传**：已有完整的文件上传API和组件支持

### 2.2 需要新增的数据表

#### 2.2.1 脑科学课程表 (`brain_science_courses`)
- 课程基本信息（课程名称、描述、目标等）
- 课程类型和分类
- 适用年龄段
- 课程时长和频次

#### 2.2.2 课程计划表 (`course_plans`)
- 学期课程安排
- 班级课程分配
- 进度计划和里程碑

#### 2.2.3 课程进度记录表 (`course_progress`)
- 班级维度的课程完成情况
- 学生个人的达标记录
- 进度百分比和达标状态
- **新增字段**：
  - `has_class_media` (BOOLEAN) - 是否有班级教学图片/视频
  - `class_media_count` (INT) - 班级媒体文件数量
  - `completion_status` (ENUM) - 完成状态：'not_started', 'in_progress', 'completed'
  - `teacher_confirmed` (BOOLEAN) - 教师是否确认完成上课
  - `media_upload_required` (BOOLEAN) - 是否要求上传媒体文件

#### 2.2.4 教学媒体记录表 (`teaching_media_records`) **新增**
- `id` (PRIMARY KEY) - 记录ID
- `class_id` (FOREIGN KEY) - 关联班级ID
- `course_progress_id` (FOREIGN KEY) - 关联课程进度ID
- `media_type` (ENUM) - 媒体类型：'class_photo', 'class_video', 'student_photo', 'student_video'
- `file_storage_id` (FOREIGN KEY) - 关联文件存储ID
- `student_id` (FOREIGN KEY, NULLABLE) - 学生ID（学员媒体时使用）
- `upload_by` (FOREIGN KEY) - 上传者ID（教师）
- `upload_time` (TIMESTAMP) - 上传时间
- `description` (TEXT) - 媒体描述
- `is_featured` (BOOLEAN) - 是否为精选媒体
- `status` (ENUM) - 状态：'active', 'archived', 'deleted'

#### 2.2.5 户外训练记录表 (`outdoor_training_records`)
- 训练活动记录
- 班级参与情况
- 完成度统计（按16周计算）
- **新增媒体支持字段**：
  - `has_media` (BOOLEAN) - 是否有媒体记录
  - `media_count` (INT) - 媒体文件数量

#### 2.2.6 离园展示记录表 (`departure_display_records`)
- 展示活动记录
- 班级执行情况
- 完成度统计（按16周计算）
- **新增媒体支持字段**：
  - `has_media` (BOOLEAN) - 是否有媒体记录
  - `media_count` (INT) - 媒体文件数量

#### 2.2.7 校外展示记录表 (`external_display_records`)
- 外出活动记录
- 累计外出次数统计
- 班级参与情况
- **新增媒体支持字段**：
  - `has_media` (BOOLEAN) - 是否有媒体记录
  - `media_count` (INT) - 媒体文件数量

#### 2.2.8 全员锦标赛记录表 (`championship_records`)
- 锦标赛活动记录
- 学期和总计统计
- 各项达标率汇总
- **新增媒体支持字段**：
  - `has_media` (BOOLEAN) - 是否有媒体记录
  - `media_count` (INT) - 媒体文件数量

## 3. 页面结构设计

### 3.1 整体布局
- 采用 `CenterContainer` 组件作为主容器
- 4个标签页：课程计划、户外训练与展示、校外展示、全员锦标赛
- 每个标签页包含统计卡片和数据列表
- **新增媒体状态指示器**：在列表中显示是否有图片/视频的状态图标

### 3.2 标签页详细设计

#### 3.2.1 课程计划标签页
**统计卡片区域：**
- 全员普及进度卡片：显示整体课程完成进度
- 结果达标率卡片：显示全体学生的达标比例

**列表区域：**
- 班级达标率列表：显示每个班级的课程达标情况
- **新增媒体状态列**：
  - 📷 图片状态图标：绿色表示有图片，灰色表示无图片
  - 🎥 视频状态图标：绿色表示有视频，灰色表示无视频
  - 媒体数量显示：如 "3张图片 2个视频"
- **新增教师确认状态**：✅ 已确认上课 / ⏳ 待确认
- 支持点击班级查看该班级每个孩子的详细达标情况和学习照片/视频
- 列表字段：班级名称、达标人数/总人数、达标率百分比、媒体状态、教师确认状态

#### 3.2.2 户外训练与展示标签页
**统计卡片区域：**
- 户外训练平均完成率卡片：显示所有班级的平均完成情况（如 3/16）
- 离园展示平均完成率卡片：显示所有班级的平均展示完成情况

**列表区域：**
- 班级执行情况列表：显示各班级的具体完成度
- 切换按钮：可在"户外训练"和"离园展示"两种视图间切换
- **新增媒体状态显示**：每行显示媒体状态图标
- 列表字段：班级名称、完成次数/总次数（如 13/16）、完成率、进度条、媒体状态图标

#### 3.2.3 校外展示标签页
**统计卡片区域：**
- 平均达标率卡片：显示校外展示的整体达标情况
- 本学期累计外出卡片：显示本学期的外出活动总次数
- 总计累计外出卡片：显示历史总的外出活动次数

**列表区域：**
- 外出活动日志列表：累加式显示各班级的外出记录
- **新增媒体状态显示**：显示每次外出活动的媒体记录状态
- 列表字段：班级名称、本学期外出次数、历史总外出次数、最近外出时间、媒体状态

#### 3.2.4 全员锦标赛标签页
**统计卡片区域：**
- 本学期神童计划达标率卡片
- 本学期课程内容达标率卡片  
- 本学期户外训练和离园展示达标率卡片
- 外出活动达标率卡片

**基本信息区域：**
- 锦标赛频次显示：每学期1次/总计X次
- 完成状态指示器
- **新增媒体汇总显示**：显示锦标赛相关的所有媒体记录统计

## 4. 媒体管理功能设计

### 4.1 媒体状态图标系统
**图标设计：**
- 📷 **图片图标**：
  - 🟢 绿色：有图片记录
  - ⚪ 灰色：无图片记录
  - 数字徽章：显示图片数量
- 🎥 **视频图标**：
  - 🟢 绿色：有视频记录
  - ⚪ 灰色：无视频记录
  - 数字徽章：显示视频数量

**状态组合显示：**
- `📷3 🎥1` - 3张图片，1个视频
- `📷0 🎥0` - 无媒体记录
- `📷5 🎥0` - 仅有图片记录

### 4.2 教师操作界面
**上课状态更新：**
- 教师可以标记课程为"已完成"状态
- 可选择上传班级教学图片/视频
- 可选择上传学员学习图片/视频
- 支持批量上传和单个上传

**媒体上传功能：**
- 复用现有的 `ElementImageUploader` 组件
- 支持图片格式：JPG, PNG, GIF, WebP
- 支持视频格式：MP4, AVI, MOV
- 文件大小限制：图片5MB，视频50MB
- 支持拖拽上传和点击上传

### 4.3 园长监督界面
**媒体状态一览：**
- 在列表中通过图标快速识别哪些班级有媒体记录
- 点击媒体图标可以预览图片/视频
- 支持媒体文件的查看和管理

**监督功能：**
- 查看教师是否真实上课（通过媒体记录验证）
- 查看学员是否完成学习（通过学员媒体记录验证）
- 统计各班级的媒体上传率和完成率

## 5. 数据关联关系

### 5.1 核心关联
- 所有教学数据都与 `classes` 表关联，通过班级维度进行统计
- 学生个人数据通过 `students` 表关联到具体班级
- 教师数据通过 `class_teachers` 关联表连接到班级

### 5.2 统计计算逻辑
- **达标率计算**：(达标学生数 / 班级总学生数) × 100%
- **平均完成率**：所有班级完成率的算术平均值
- **16周进度**：按学期16周为基准，计算完成的周次比例
- **累计统计**：跨学期的历史数据累加

## 7. API接口设计

### 7.1 数据获取接口
- `GET /api/teaching-center/course-progress` - 获取课程进度统计（包含媒体状态）
- `GET /api/teaching-center/outdoor-training` - 获取户外训练数据（包含媒体状态）
- `GET /api/teaching-center/departure-display` - 获取离园展示数据（包含媒体状态）
- `GET /api/teaching-center/external-display` - 获取校外展示数据（包含媒体状态）
- `GET /api/teaching-center/championship` - 获取锦标赛数据（包含媒体状态）

### 7.2 媒体管理接口
- `POST /api/teaching-center/upload-media` - 上传教学媒体文件
- `GET /api/teaching-center/media/{recordId}` - 获取特定记录的媒体文件列表
- `DELETE /api/teaching-center/media/{mediaId}` - 删除媒体文件
- `PUT /api/teaching-center/media/{mediaId}` - 更新媒体文件信息

### 7.3 状态更新接口
- `PUT /api/teaching-center/confirm-completion` - 教师确认完成上课
- `GET /api/teaching-center/media-status/{classId}` - 获取班级媒体状态统计

## 8. 权限和角色

### 8.1 访问权限
- **管理员**：完整的查看和管理权限，包括所有媒体文件
- **园长**：查看所有数据和媒体文件，监督教学质量
- **教师**：查看自己班级的数据，上传班级和学员媒体文件

### 8.2 媒体权限控制
- **教师权限**：
  - 只能上传自己负责班级的媒体文件
  - 只能查看自己班级的媒体记录
  - 可以删除自己上传的媒体文件
- **园长权限**：
  - 可以查看所有班级的媒体文件
  - 可以监督媒体上传情况
  - 不能删除教师上传的媒体文件
- **数据安全**：
  - 媒体文件访问需要身份验证
  - 支持媒体文件的访问日志记录

## 9. 技术实现要点

### 9.1 文件存储集成
- 复用现有的 `FileStorage` 模型和服务
- 通过 `module` 字段标识为 'teaching-center'
- 通过 `referenceType` 和 `referenceId` 关联到具体的教学记录

### 9.2 媒体状态缓存
- 在课程进度记录中缓存媒体统计信息
- 避免每次查询都统计媒体文件数量
- 在媒体文件增删时更新缓存

### 9.3 性能优化
- 媒体文件列表支持分页加载
- 图片支持缩略图预览
- 视频支持封面图生成
- 支持媒体文件的懒加载

## 12. 用户体验设计

### 12.1 教师使用流程
1. 教师完成课程教学后，进入教学中心

3. 可选择上传班级教学图片/视频

### 12.2 园长监督流程
1. 园长进入教学中心查看整体进度

3. 点击媒体图标预览教学和学习情况
4. 对没有媒体记录的班级进行重点关注
5. 通过数据统计了解整体教学质量

## 13. 数据库设计详细说明

### 13.1 表结构设计

#### 脑科学课程表 (`brain_science_courses`)
```sql
CREATE TABLE brain_science_courses (
  id INT PRIMARY KEY AUTO_INCREMENT,
  course_name VARCHAR(100) NOT NULL COMMENT '课程名称',
  course_description TEXT COMMENT '课程描述',
  course_type ENUM('core', 'extended', 'special') DEFAULT 'core' COMMENT '课程类型',
  target_age_min INT COMMENT '适用最小年龄（月）',
  target_age_max INT COMMENT '适用最大年龄（月）',
  duration_minutes INT COMMENT '课程时长（分钟）',
  frequency_per_week INT DEFAULT 1 COMMENT '每周频次',
  objectives JSON COMMENT '课程目标',
  materials JSON COMMENT '所需材料',
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### 课程计划表 (`course_plans`)
```sql
CREATE TABLE course_plans (
  id INT PRIMARY KEY AUTO_INCREMENT,
  course_id INT NOT NULL,
  class_id INT NOT NULL,
  semester VARCHAR(20) NOT NULL COMMENT '学期',
  academic_year VARCHAR(10) NOT NULL COMMENT '学年',
  planned_start_date DATE COMMENT '计划开始日期',
  planned_end_date DATE COMMENT '计划结束日期',
  total_sessions INT DEFAULT 16 COMMENT '总课时数',
  completed_sessions INT DEFAULT 0 COMMENT '已完成课时',
  plan_status ENUM('draft', 'active', 'completed', 'cancelled') DEFAULT 'draft',
  created_by INT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (course_id) REFERENCES brain_science_courses(id),
  FOREIGN KEY (class_id) REFERENCES classes(id),
  FOREIGN KEY (created_by) REFERENCES users(id)
);
```

#### 课程进度记录表 (`course_progress`)
```sql
CREATE TABLE course_progress (
  id INT PRIMARY KEY AUTO_INCREMENT,
  course_plan_id INT NOT NULL,
  class_id INT NOT NULL,
  session_number INT NOT NULL COMMENT '第几课时',
  session_date DATE COMMENT '上课日期',
  completion_status ENUM('not_started', 'in_progress', 'completed') DEFAULT 'not_started',
  teacher_confirmed BOOLEAN DEFAULT FALSE COMMENT '教师确认完成',
  attendance_count INT DEFAULT 0 COMMENT '出勤人数',
  target_achieved_count INT DEFAULT 0 COMMENT '达标人数',
  achievement_rate DECIMAL(5,2) DEFAULT 0.00 COMMENT '达标率',
  has_class_media BOOLEAN DEFAULT FALSE COMMENT '是否有班级媒体',
  class_media_count INT DEFAULT 0 COMMENT '班级媒体数量',
  media_upload_required BOOLEAN DEFAULT FALSE COMMENT '是否要求上传媒体',
  notes TEXT COMMENT '备注',
  teacher_id INT COMMENT '授课教师',
  confirmed_at TIMESTAMP NULL COMMENT '确认时间',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (course_plan_id) REFERENCES course_plans(id),
  FOREIGN KEY (class_id) REFERENCES classes(id),
  FOREIGN KEY (teacher_id) REFERENCES teachers(id)
);
```

#### 教学媒体记录表 (`teaching_media_records`)
```sql
CREATE TABLE teaching_media_records (
  id INT PRIMARY KEY AUTO_INCREMENT,
  class_id INT NOT NULL,
  course_progress_id INT NOT NULL,
  media_type ENUM('class_photo', 'class_video', 'student_photo', 'student_video') NOT NULL,
  file_storage_id INT NOT NULL,
  student_id INT NULL COMMENT '学员ID（学员媒体时使用）',
  upload_by INT NOT NULL COMMENT '上传者ID',
  upload_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  description TEXT COMMENT '媒体描述',
  is_featured BOOLEAN DEFAULT FALSE COMMENT '是否为精选媒体',
  status ENUM('active', 'archived', 'deleted') DEFAULT 'active',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (class_id) REFERENCES classes(id),
  FOREIGN KEY (course_progress_id) REFERENCES course_progress(id),
  FOREIGN KEY (file_storage_id) REFERENCES file_storages(id),
  FOREIGN KEY (student_id) REFERENCES students(id),
  FOREIGN KEY (upload_by) REFERENCES users(id)
);
```

### 13.2 索引设计
```sql
-- 课程进度查询优化
CREATE INDEX idx_course_progress_class_date ON course_progress(class_id, session_date);
```
