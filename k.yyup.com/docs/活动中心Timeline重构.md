# 活动中心Timeline重构文档

## 📋 项目概述

**重构目标**：将活动中心从传统的标签页模式改造为业务中心的Timeline队列模式，清晰展示活动管理的完整流程，方便园长一目了然地掌握活动进展。

**重构时间**：2025-10-05

**负责人**：AI Assistant

---

## 🎯 重构背景

### 原有问题
1. **功能分散**：活动管理功能分散在多个标签页中，缺乏整体视图
2. **流程不清晰**：园长难以快速了解活动从策划到分析的完整流程
3. **营销配置隐藏**：团购、积攒等营销功能在创建流程中，不够突出
4. **数据孤立**：各个环节的数据统计分散，缺乏关联性

### 业务需求
1. **流程可视化**：将活动管理的8个核心环节以Timeline形式展示
2. **进度实时化**：每个环节显示实时进度百分比（基于真实数据计算）
3. **营销突出化**：将营销配置（团购、积攒）作为核心功能展示
4. **页面生成化**：强调最终生成的是活动报名页面，而不是简单的海报

---

## 📊 Timeline流程详解

### 1. 活动策划 (85%进度)

**业务内容**：
- 选择活动模板
- 填写基本信息（标题、时间、地点、人数）
- 设置活动类型和目标

**数据统计**：
- `totalActivities`: 总活动数
- `totalTemplates`: 活动模板总数
- `usedTemplates`: 已使用模板数
- `draftActivities`: 草稿活动数

**进度计算**：
```typescript
progress = Math.min((totalActivities / 100) * 100, 100)
```

**快捷操作**：
- 新建活动
- 查看模板
- AI策划

---

### 2. 内容制作 (72%进度) ⭐ 核心流程

**业务内容**：
- 🎨 **海报设计**：选择模板、编辑海报、AI生成海报
- 🛒 **营销配置**：设置团购、积攒、优惠券、推荐奖励

**营销配置详解**：

#### 团购活动
- 团购人数设置（2-50人）
- 团购价格设置
- 团购时限设置

#### 积攒活动
- 积攒目标设置（10-1000人）
- 积攒奖励类型（折扣/礼品/免费）
- 奖励规则设置

#### 优惠券
- 优惠券类型（满减/折扣）
- 发放数量
- 使用条件

#### 推荐奖励
- 推荐奖励金额
- 最大奖励次数

**数据统计**：
- `totalPosters`: 海报总数
- `activitiesWithMarketing`: 启用营销的活动数
- `publishedActivities`: 已发布活动数
- `draftActivities`: 草稿活动数

**进度计算**：
```typescript
progress = totalActivities > 0 ? (publishedActivities / totalActivities) * 100 : 0
```

**快捷操作**：
- 设计海报
- 营销配置
- 预览效果

```typescript
async getTimeline() {
  // 1. 并行查询所有统计数据
  const [
    totalActivities,
    publishedActivities,
    totalRegistrations,
    totalEvaluations,
    // ... 其他统计数据
  ] = await Promise.all([
    Activity.count({ where: { deletedAt: null } }),
    Activity.count({ where: { deletedAt: null, publishStatus: 'published' } }),
    // ... 其他查询
  ]);

  // 2. 构建Timeline数据
  const timeline = [
    {
      id: 'activity-planning',
      title: '活动策划',
      description: '模板选择和基本信息设置',
      icon: 'Lightbulb',
      status: totalActivities > 0 ? 'completed' : 'pending',
      progress: Math.min((totalActivities / 100) * 100, 100),
      stats: { /* 统计数据 */ },
      actions: [ /* 快捷操作 */ ]
    },
    // ... 其他流程
  ];

  return { success: true, data: timeline };
}
```

#### 2. Controller层

**文件**：`server/src/controllers/centers/activity-center.controller.ts`

**核心方法**：`getTimeline()`

```typescript
static async getTimeline(req: Request, res: Response) {
  try {
    const activityCenterService = new ActivityCenterService();
    const result = await activityCenterService.getTimeline();
    return ApiResponse.success(res, result.data, '活动中心Timeline数据获取成功');
  } catch (error) {
    console.error('❌ 获取活动中心Timeline数据失败:', error);
    return res.status(500).json({
      success: false,
      error: { code: 'TIMELINE_ERROR', message: '获取Timeline数据失败' }
    });
  }
}
```

#### 3. Route层

**文件**：`server/src/routes/centers/activity-center.routes.ts`

**路由定义**：
```typescript
router.get('/timeline', authenticate, ActivityCenterController.getTimeline);
```

**API端点**：`GET /api/centers/activity/timeline`

## 📝 数据结构

### Timeline Item数据结构

```typescript
interface TimelineItem {
  id: string;                    // 流程唯一标识
  title: string;                 // 流程标题
  description: string;           // 流程描述
  icon: string;                  // 图标名称（Lucide图标）
  status: 'completed' | 'in-progress' | 'pending';  // 状态
  progress: number;              // 进度百分比（0-100）
  stats: Record<string, any>;    // 统计数据对象
  actions: Action[];             // 快捷操作按钮数组
}

interface Action {
  key: string;                   // 操作唯一标识
  label: string;                 // 按钮文本
  type: 'primary' | 'success' | 'warning' | 'default' | 'info';  // 按钮类型
}
```
