# 幼儿园管理系统等保定级报告

**系统名称**: 幼儿园管理系统（k.yyup.cc）  
**定级结果**: **三级（监督保护级）**  
**编制日期**: 2025-12-29  
**编制单位**: [幼儿园管理系统开发团队]

---

## 一、系统基本情况

### 1.1 系统概述
- **系统名称**: 幼儿园管理系统
- **系统域名**: k.yyup.cc
- **系统类型**: 多租户SaaS云平台
- **服务对象**: 幼儿园管理者、教师、家长
- **部署方式**: 云服务器部署（Sealos平台）
- **技术架构**: Vue 3 + Express.js + MySQL

### 1.2 系统规模
- **代码规模**: ~150,000行代码
- **组件数量**: 80+ Vue组件，162+页面
- **API端点**: 155+ REST API
- **数据模型**: 73+数据模型
- **用户规模**: 支持多租户，单租户100-500人

### 1.3 业务功能
- 学生信息管理（个人档案、健康记录、人脸识别）
- 教师管理（教师档案、考勤、绩效）
- 招生管理（在线报名、面试安排、录取通知）
- 家长服务（家园沟通、缴费管理、活动报名）
- 活动中心（活动发布、报名、团购）
- 财务管理（收费、缴费、报表）

## 二、定级依据

### 2.1 法律法规依据
1. **《中华人民共和国网络安全法》**
   - 第二十一条：国家实行网络安全等级保护制度
   - 第三十一条：关键信息基础设施运营者应当履行安全保护义务

2. **《网络安全等级保护条例》（等保2.0）**
   - GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》
   - GB/T 25070-2019《信息安全技术 网络安全等级保护设计技术要求》

3. **教育行业标准**
   - 教育部《教育行业信息系统安全等级保护定级工作指南（试行）》
   - 《个人信息保护法》第二十八条：敏感个人信息处理规则
   - 《未成年人保护法》第七十二条：未成年人个人信息保护

### 2.2 定级原则
根据《信息系统安全等级保护定级指南》（GB/T 22240-2020），信息系统安全等级由以下两个方面确定：
1. **业务信息安全受到破坏时的客体受侵害程度**
2. **系统服务受到破坏时的客体受侵害程度**

定级矩阵：
| 受侵害客体 | 一般损害 | 严重损害 | 特别严重损害 |
|-----------|---------|---------|-------------|
| 公民/法人/组织合法权益 | 二级 | **三级** | 四级 |
| 社会秩序、公共利益 | 二级 | **三级** | 四级 |
| 国家安全 | 三级 | 四级 | 五级 |

---

## 三、定级分析

### 3.1 业务信息安全分析

#### 敏感信息类型及影响
| 信息类型 | 数据量级 | 敏感程度 | 破坏影响 |
|---------|---------|---------|---------|
| **未成年人个人信息** | 1000+/园 | 极高 | 严重损害未成年人权益 |
| 身份证号 | 1000+/园 | 高 | 身份信息泄露 |
| 人脸识别数据 | 1000+/园 | 高 | 生物特征泄露 |
| 健康档案 | 1000+/园 | 高 | 健康隐私泄露 |
| 家长联系方式 | 2000+/园 | 中 | 骚扰电话、诈骗风险 |
| 财务缴费记录 | 10000+/园 | 中 | 经济信息泄露 |

#### 影响分析
- **破坏后果**：系统存储大量14岁以下未成年人敏感个人信息（姓名、身份证号、人脸数据、健康档案），一旦泄露将：
  1. **严重损害未成年人权益**：身份信息可能被用于诈骗、拐卖等犯罪
  2. **影响社会稳定**：家长恐慌、媒体报道、政府问责
  3. **法律责任**：违反《个人信息保护法》《未成年人保护法》，面临高额罚款和刑事责任

**结论**：业务信息安全受到破坏对公民权益造成**严重损害**，定级建议：**三级**

---

### 3.2 系统服务安全分析

#### 服务中断影响
| 服务 | 用户规模 | 中断影响 | 恢复时长要求 |
|-----|---------|---------|-------------|
| 在线招生 | 500+家长/园 | 错过报名截止时间 | < 4小时 |
| 考勤打卡 | 100+教师/园 | 影响正常运营 | < 1小时 |
| 家园沟通 | 2000+家长/园 | 紧急通知无法送达 | < 1小时 |
| 缴费系统 | 500+家长/园 | 财务纠纷 | < 4小时 |

#### 影响分析
- **服务中断后果**：
  1. **影响教育教学秩序**：考勤、排课、评价等核心业务中断
  2. **家长投诉**：缴费、报名等服务中断引发家长不满
  3. **社会影响**：多个幼儿园同时受影响（多租户），可能引发媒体关注

**结论**：系统服务受到破坏对社会秩序造成**严重损害**，定级建议：**三级**

---

### 3.3 系统特性分析

#### 增强定级因素
1. **多租户架构**：服务多个幼儿园，破坏影响扩大
2. **互联网暴露**：公网访问，攻击面大
3. **敏感操作**：涉及未成年人信息，监管严格
4. **财务功能**：涉及资金流转，风险高
5. **生物特征**：人脸识别数据，不可逆泄露

#### 降低定级因素
- 无（无降低定级因素）

---

## 四、定级结论

### 4.1 综合评定
根据上述分析，依据《信息系统安全等级保护定级指南》：
- **业务信息安全**：破坏后对公民权益造成严重损害 → **三级**
- **系统服务安全**：破坏后对社会秩序造成严重损害 → **三级**
- **系统特性**：多租户、互联网暴露、未成年人信息 → **增强至三级**

**最终定级：三级（监督保护级）**

### 4.2 定级说明
本系统应定为**三级信息系统**，理由如下：
1. ✅ 存储处理大量未成年人（14岁以下）敏感个人信息
2. ✅ 多租户SaaS架构，破坏影响范围广
3. ✅ 涉及财务数据、健康档案、人脸识别等高敏感信息
4. ✅ 互联网公开访问，安全风险高
5. ✅ 破坏后可能严重损害未成年人权益和社会秩序

### 4.3 备案要求
根据《信息安全等级保护管理办法》第十五条：
- 三级信息系统应当在**安全保护等级确定后10个工作日内**，到所在地设区的市级以上公安机关备案
- 备案材料：
  1. 定级报告
  2. 系统拓扑图及说明
  3. 系统安全组织机构及管理制度
  4. 系统安全保护设施设计实施方案
  5. 系统使用的技术产品清单

## 五、后续工作

### 5.1 等保测评
- **测评周期**：每年至少进行一次等级测评
- **测评机构**：由具备资质的等级测评机构实施
- **测评内容**：技术要求（安全物理环境、安全通信网络、安全区域边界、安全计算环境、安全管理中心）和管理要求（安全管理制度、安全管理机构、安全管理人员、安全建设管理、安全运维管理）

### 5.2 整改要求
- 根据等保三级基本要求，实施技术和管理整改
- 重点：数据加密、访问控制、安全审计、应急响应
- 整改期限：90天（等保测评前完成）

### 5.3 持续监管
- 建立安全管理组织
- 定期开展安全培训
- 每年进行安全自查
- 每年进行等保测评
