# 移动端开发计划

## 项目概述
使用 Vant4 组件库为客户端开发移动端版本，对应 PC 端目录结构。

### 目标目录
- `client/src/pages/centers` → `client/src/pages/mobile/centers`
- `client/src/pages/teacher-center` → `client/src/pages/mobile/teacher-center`
- `client/src/pages/parent-center` → `client/src/pages/mobile/parent-center`

### 技术栈
- **UI 框架**: Vant 4 (移动端 UI 组件库)
- **开发语言**: TypeScript
- **路由**: Vue Router
- **状态管理**: Pinia
- **样式**: SCSS

### 2. 组件迁移原则
- 保持与 PC 端相同的业务逻辑
- 使用 Vant4 组件替代 Element Plus
- 适配移动端交互习惯（点击、滑动、触摸）
- 响应式布局优化
- 减少页面复杂度，简化操作流程

### 3. Vant4 组件选择

#### 基础组件
- **van-cell**: 单元格（替代 el-table-row 中的行）
- **van-card**: 卡片（展示信息块）
- **van-button**: 按钮（所有按钮操作）
- **van-nav-bar**: 导航栏（页面头部）
- **van-tabbar**: 底部导航栏（主导航）
- **van-sidebar**: 侧边栏（辅助导航）

#### 表单组件
- **van-field**: 输入框（表单输入）
- **van-picker**: 选择器（下拉选择）
- **van-datetime-picker**: 日期时间选择器
- **van-radio**: 单选框
- **van-checkbox**: 复选框
- **van-switch**: 开关

#### 反馈组件
- **van-toast**: 轻提示（消息反馈）
- **van-dialog**: 弹出框（确认操作）
- **van-loading**: 加载（异步操作）
- **van-empty**: 空状态（无数据）

#### 业务组件
- **van-list**: 列表（长列表展示）
- **van-pull-refresh**: 下拉刷新
- **van-index-bar**: 索引栏（快速定位）
- **van-steps**: 步骤条（流程展示）
- **van-collapse**: 折叠面板（分组展示）
- **van-badge**: 徽章（提醒数量）

## 开发注意事项

### 1. 兼容性
- 支持 iOS 10+ 和 Android 5.0+
- 适配不同屏幕尺寸（使用 Vant 的响应式方案）
- 处理不同浏览器的兼容性问题

### 2. 性能优化
- 路由懒加载
- 图片懒加载
- 长列表虚拟滚动
- 减少页面重绘

### 3. 用户体验
- 简化操作流程
- 提供清晰的视觉反馈
- 优化加载状态
- 处理网络异常

### 4. 状态管理
- 使用 Pinia 管理全局状态
- 页面间数据共享
- 用户登录状态保持
