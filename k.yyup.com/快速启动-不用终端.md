# 🚀 快速启动指南（避免Cursor终端卡死）

## 问题说明
Cursor的集成终端确实经常卡死，这是软件本身的问题。

## 🎯 解决方案

### 方法1：双击批处理文件（最简单）

我已经创建了3个启动脚本：

1. **`start-all.bat`** - 一键启动前后端（推荐）
2. **`start-backend.bat`** - 只启动后端服务
3. **`start-frontend.bat`** - 只启动前端服务

**使用方法：**
- 双击 `start-all.bat` 即可自动启动整个系统
- 会自动打开两个窗口：后端(3000端口) + 前端(5173端口)

### 方法2：使用Windows终端

```bash
# 方法A：Windows PowerShell
Win + R → 输入 powershell → 回车

# 方法B：命令提示符  
Win + R → 输入 cmd → 回车

# 方法C：Windows Terminal
Win + X → 选择 "Windows Terminal"
```

### 方法3：VSCode终端（如果有安装）

VSCode的终端比Cursor稳定很多，可以作为备用。

## 🌐 访问地址

启动成功后：
- **前端应用**: http://localhost:5173
- **后端API**: http://localhost:3000

## ⚡ 快速测试

1. 双击 `start-all.bat`
2. 等待两个服务启动（大约30-60秒）
3. 浏览器访问 http://localhost:5173

## 🛠️ 如果遇到端口冲突

端口被占用时的解决方法：

```bash
# 查看端口占用
netstat -ano | findstr :5173
netstat -ano | findstr :3000

# 结束进程（替换PID）
taskkill /pid <进程ID> /f
```
