## 测试须知

### 1. 登录要求
- 需要使用 **parent** 角色账号登录
- 登录后确保权限已加载

### 2. 检测流程
```
每个页面执行以下步骤：
1. 使用 Task 工具开启子代理
2. 子代理使用 browser_navigate 导航到页面
3. 使用 browser_wait_for({ time: 3 }) 等待加载
4. 使用 browser_snapshot 获取页面快照
5. 检测页面元素（按钮、输入框、卡片、表格）
6. 使用 browser_click 点击交互元素（限制5个以内）
7. 使用 browser_console_messages({ level: 'error' }) 获取错误
8. 记录检测结果
9. 销毁子代理
10. 更新任务状态
```

### 3. 状态标记规则
- 【未开始】- 页面还未开始检测
- 【进行中】- 正在检测该页面
- 【已完成】- 检测完成，无重大问题

### 4. 问题判定标准
- **严重(high)**: 页面无法渲染、路由404、控制台JavaScript错误
- **中等(medium)**: API调用失败、部分功能异常
- **轻微(low)**: 警告信息、权限提示
