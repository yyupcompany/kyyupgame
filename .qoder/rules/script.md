# 脚本使用规范

## 重要原则
- **仅用于检测**：脚本只能用于检测和分析代码，不允许修改代码
- **只读操作**：所有脚本操作必须是只读的，不得对源代码进行任何写入或修改
- **报告问题**：脚本应该只报告发现的问题、错误或需要改进的地方，而不是自动修复它们

## 允许的操作
- 代码静态分析
- 语法检查
- 代码质量检测
- 依赖关系分析
- 性能分析
- 安全漏洞扫描
- 生成检测报告

## 禁止的操作
- ❌ 自动修改源代码
- ❌ 自动格式化代码
- ❌ 自动重构代码
- ❌ 写入或覆盖文件
- ❌ 任何形式的代码变更

## 示例场景
### ✅ 正确使用
# 检测代码质量
eslint src/ --format json

# 分析依赖
npm audit

# 类型检查
tsc --noEmit

### ❌ 错误使用
# 不允许自动修复
eslint src/ --fix

# 不允许自动格式化
prettier --write src/

# 不允许自动重构
jscodeshift -t transform.js src/
